E-Learning Ontology Documentation

1. Abstract

The E-Learning Ontology is developed for a university-style online course portal. It models the core activities of an online learning environment, including system users (students and instructors), academic materials (courses, modules, and learning objectives), and the performance evaluation process (assignments, submissions, grades, and feedback text). Built using Protégé and checked for logical issues via the HermiT reasoner, the ontology serves as a structural blueprint to tie grades directly back to submitted coursework, assignments, and specific instructors.

5. Classes

This section lists each class from the hierarchy along with its parent class and a short description of its exact role in our project:

AcademicContent

Parent Class: owl:Thing
Short Description: A general container used to group all structured course materials, including separate study blocks and specific learning goals.

Assessment

Parent Class: owl:Thing
Short Description: A general category representing any method or academic task used to check a student's knowledge level.

Assignment

Parent Class: Assessment
Short Description: A specific graded homework task or project configured inside a course module that must be completed by students.

Course

Parent Class: AcademicContent
Short Description: A full university course containing sequential modules, learning goals, assigned tasks, and managed by a single main instructor.

Evaluation

Parent Class: owl:Thing
Short Description: A generalized class tracking the performance assessment of student work, splitting into numeric scores and written reviews.

Feedback

Parent Class: Evaluation
Short Description: Alphanumeric critique and commentary written by an instructor to review a student's file upload.

Grade

Parent Class: Evaluation
Short Description: The exact numerical points awarded to a student for their coursework submission.

Instructor

Parent Class: Person
Short Description: A university faculty member responsible for creating course material, guiding modules, and scoring student work.

LateSubmission

Parent Class: Submission
Short Description: A coursework file uploaded by a student that has an internal date value occurring after the set assignment deadline.

LearningObjective

Parent Class: AcademicContent
Short Description: A specific milestone or core skill that a student is expected to master after finishing a module or full course.

Module

Parent Class: AcademicContent
Short Description: A single structural chapter or weekly topic block used to group learning materials inside a course.

Person

Parent Class: owl:Thing
Short Description: A top-level category used to hold personal details for any user interacting with the educational portal.

Student

Parent Class: Person
Short Description: A university user enrolled in specific courses who uploads project files to satisfy task deadlines.

Submission

Parent Class: owl:Thing
Short Description: An archive file or document turned in by a student to fulfill an assigned coursework task.

6. Object Properties

The relationships linking individuals of different classes together are outlined in the structural table below:

Property Domain Range Description
enrolledIn Student Course Connects a student user to the active courses they are taking.
hasAssignment Course Assignment Links a full course to the specific assignments that need to be completed.
hasFeedback Submission Feedback Connects an uploaded coursework file to the evaluation notes left by a teacher.
hasGrade Submission Grade Links a student's submission to its final numeric score instance. Functional property type (maximum of 1 grade per submission).
hasLearningObjective Course LearningObjective Maps a course to the explicit concepts or skill milestones students should acquire.
hasModule Course Module Links a full course to its chronological learning units or weekly modules.
hasPrerequisite Course Course Defines dependencies between classes by showing if a course requires completing a prior course. This property is transitive.
hasStudent Course Student Maps an active course back to its registered students. Configured as the direct inverse of enrolledIn.
hasSubmission Assignment Submission Connects a graded task to all the individual student files turned in for it.
isAssignmentOf Assignment Course Identifies which parent course an assignment individual belongs to. Inverse of hasAssignment.
isFeedbackFor Feedback Submission Points a written feedback review back to the exact submission it evaluates. Inverse of hasFeedback.
isLearningObjectiveOf LearningObjective Course Links a specific skill objective back to its source course. Inverse of hasLearningObjective.
isModuleOf Module Course Links a weekly chapter instance back to its parent course. Inverse of hasModule.
isTaughtBy Course Instructor Links a course to its main teacher. Functional property type (each course has exactly 1 primary instructor).
submissionFor Submission Assignment Connects a student's uploaded solution back to the original assignment prompt. Inverse of hasSubmission.
submits Student Submission Links a student to the file individuals they have uploaded to the platform.
submittedBy Submission Student Traces a turned-in file back to the student individual who uploaded it. Inverse of submits.
teaches Instructor Course Connects a faculty member to the courses they manage. Inverse of isTaughtBy.

7. Data Properties

The data properties used to add text descriptions, dates, and numbers directly to class instances are detailed below:

Property Domain Datatype Description
assignmentTitle Assignment xsd:string The official descriptive name given to an assignment.
courseCode Course xsd:string The university code tag used to identify a class (e.g., "CS-550"). Functional type.
courseTitle Course xsd:string The complete structural name of a course (e.g., "Knowledge Engineering").
dueDate Assignment xsd:dateTime The exact calendar date and hour cut-off time for an assignment submission.
employeeID Instructor xsd:string The unique staff identification number assigned to an instructor. Functional type.
feedbackText Feedback xsd:string The actual written comments left by an instructor reviewing turned-in work.
hasEmail Person xsd:string The official university email address connected to a specific system user.
hasName Person xsd:string The standard first and last name string identifying a student or instructor.
maxScore Assignment xsd:float The maximum points possible for a task (e.g., 100.0). Functional type.
moduleDescription Module xsd:string A short paragraph summarizing what subject matter is covered inside a module.
moduleOrder Module xsd:integer The specific sequence number for a module (e.g., 1, 2, 3 to represent week ordering). Functional type.
moduleTitle Module xsd:string The specific topic header given to a single course module.
score Grade xsd:float The exact numerical grade score earned by a student on a submission. Functional type.
studentID Student xsd:string The official university registration card number for a student. Functional type.
submissionDate Submission xsd:dateTime The exact log timestamp recorded when a student uploads their solution file. Functional type.
